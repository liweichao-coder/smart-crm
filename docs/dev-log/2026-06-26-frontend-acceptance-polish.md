# 2026-06-26 前端验收与演示布局优化

## 背景

本地验收 Smart CRM Web 端时，重点检查登录、仪表盘、AI 副驾、智能录单、订单、销售报表、团队成员和客户页的控制台错误、横向溢出和首屏观感。

## 发现问题

- AI 副驾页的 LLM 摘要内容较长时，会把右侧摘要卡撑高，导致指标卡和评分列表被推到很靠后的区域，影响答辩首屏观感。
- 客户页宽表会撑开页面整体宽度，右侧操作按钮出现在视口外，形成页面级横向滚动。

## 更新内容

- AI 副驾页的长摘要卡增加独立 `crm-copilot-summary--scrollable` 样式，摘要正文在卡片内部滚动，保留首屏核心指标。
- AI 文本块增加长文本换行和更自然的段落换行能力，避免 LLM 输出中的长句破坏布局。
- 通用表格外层 `crm-table-wrap` 增加横向滚动约束，宽表只在表格区域内滚动，不再撑宽整个页面。

## 验证

- 浏览器验收：登录、组织选择、仪表盘、AI 副驾、智能录单、订单、销售报表、团队成员、客户页均无页面控制台错误。
- AI 副驾页：长 LLM 摘要不再撑爆 hero，指标卡回到首屏可见范围。
- 客户页：页面级横向滚动消失，宽表在表格容器内滚动。
- `npm run lint` 通过。
- `npm test` 通过，51 个前端单元测试全部通过。
- `npm run build` 通过。
