# 2026-06-26 Chrome 视觉巡检与中宽布局优化

## 背景

使用用户当前 Chrome 标签页验收 Smart CRM Web 端，在约 1059px 宽窗口下检查商机、AI 副驾、智能录单、订单、客户、销售报表和登录页的控制台错误、卡片排版、字体排版和整体观感。

## 发现问题

- 中等宽度窗口下左侧导航仍保持桌面宽度，右侧内容区域变窄，资源页头和工具栏显得过高、过空。
- 客户页字段选择器原本挤在右侧窄栏中，排版不够像后台系统。
- 订单和报表指标卡内的标签、数值和说明没有明确分行，数字与说明容易贴在一起。
- AI 副驾页摘要已能滚动，但中宽下摘要区域偏窄，长文本可读性不足。

## 更新内容

- 为 `.crm-content` 增加容器查询，使资源页头、工具栏、指标卡和 AI 副驾摘要能基于主内容宽度自适应。
- 资源页头改为紧凑网格布局：标题靠左，操作按钮自动换行，减少首屏空白。
- 工具栏改为搜索框在上、字段选择器整行铺开、状态徽章横向换行的布局。
- 指标卡文本明确分成标签、数值、说明三段，避免数字和说明粘连。
- AI 副驾在中宽内容区下改为标题与摘要上下布局，摘要使用整行宽度展示。

## 验证

- Chrome 标签页视觉复查：商机、AI 副驾、订单、客户、销售报表页面无控制台错误。
- 商机/客户页工具栏在中宽窗口下不再右侧挤压，字段选择与状态徽章更清晰。
- 订单/报表指标卡数值与说明分行显示。
- `npm run lint` 通过。
- `npm test` 通过，51 个前端单元测试全部通过。
- `npm run build` 通过。
