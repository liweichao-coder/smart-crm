# 2026-06-23 Real Contact Master Data

## 背景

客户和联系人是 CRM 的主数据根节点。此前前端快速创建为了降低演示成本，会在客户行业、邮箱和联系人邮箱等字段上使用默认演示值，容易让“真实可用”看起来像半成品录入。本轮将客户、联系人表单补齐为更接近真实 CRM 的主数据录入。

## 本次改动

- 客户创建/编辑表单新增联系人、电话、邮箱、城市、来源和客户等级字段。
- 联系人创建/编辑表单新增电话字段。
- 客户等级、客户状态、联系人状态改为下拉选择，避免用户手输枚举导致 422。
- 表单输入支持 `email`、`tel`、`select` 等类型，批量编辑也支持下拉选择。
- 批量编辑草稿尊重列级默认值，启用下拉字段后不再出现界面显示默认项但实际提交空值的问题。
- 新增 `src/payloadUtils.js`，将客户/联系人 payload 构造从页面中抽出，避免前端静默写入固定演示邮箱或“待补充”行业。
- `createDraftFromColumns()` 支持列级 `defaultValue`，客户/联系人关键字段默认空白，由用户真实填写。
