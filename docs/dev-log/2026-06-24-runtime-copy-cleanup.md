# 2026-06-24 运行时占位文案收口

## 背景

在最终演示前做了一轮 `TODO/mock/占位/待补充` 审计。大多数命中项来自历史迭代日志、测试说明或输入框 placeholder，不影响当前运行行为。仍有少量后端运行时通知文案会显示“待补充”，容易在答辩时被误解为未完成占位。

## 改动

- 客户互动转任务描述中，缺失互动结果时显示“未填写”，不再显示“待补充”。
- 订单审批通知中，SLA 未计算出剩余时间时显示“SLA 未设置”，不再显示“SLA 待补充”。
- 将演示种子数据中的合同跟进结果从“合同待补充”改为“条款进入法务复核”，保持业务语义但避免像模板占位。

## 验收点

- 该改动不改变数据库模型、权限、审批或通知聚合逻辑。
- 保留历史日志里的“待补充”描述作为早期问题和迭代证据，不回写旧记录。
