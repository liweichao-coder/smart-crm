# 2026-06-26 CordysCRM 参考与课堂演示范围收口

## 背景

- 课程项目需要更像一个聚焦的 CRM 演示产品，而不是把所有后台功能平铺在侧边栏。
- 参考 CordysCRM 的产品结构时，只吸收 L2C 主链路、工作台入口、CRM Skills 问答和 BI 复盘这类产品思路。
- Smart CRM 仍保持自研实现，不使用 CordysCRM 源码、商标或受保护素材。
- 本轮新增详细执行计划：[docs/plans/2026-06-26-cordyscrm-reference-course-demo-plan.md](../plans/2026-06-26-cordyscrm-reference-course-demo-plan.md)，后续在主会话内逐步执行，不再新开子智能体。

## 本次调整

- 保留 Smart CRM 自有 React + FastAPI + SQLite 技术栈，不引入 CordysCRM 源码和商标资源。
- 左侧导航改为“演示主线”和“AI 亮点”优先：
  - 演示工作台
  - 客户管理
  - 销售线索
  - 商机跟进
  - 订单管理
  - AI 销售副驾
  - AI 智能录单
  - 销售报表
- 侧栏信息架构进一步收敛为“演示主线 / AI 亮点 / 经营分析 / 更多功能”，避免把销售报表混在基础业务流里。
- 低频后台功能收进“更多功能”，包括联系人、商品、任务、销售目标、工单、团队成员、权限矩阵和审计页面。
- 工作台首页突出 L2C 主链路、AI 销售副驾、智能录单和快捷入口。
- 课堂主展示路径统一为：登录 -> 工作台 -> 客户 -> 线索/商机 -> AI 副驾 -> 智能录单 -> 订单 -> 销售报表。
- 验收说明统一强调“简单 CRM 主流程 + AI 亮点 + 真实后端闭环”，避免把项目描述成覆盖所有场景的大型 CRM。
- 前端壳层补充：
  - 工作台顶部加入“客户 -> 线索 -> 商机 -> 订单”的可点击 L2C 路线卡。
  - 快捷入口扩展为客户、线索、商机、订单、AI 销售副驾、AI 智能录单 6 个演示动作。
  - 增加“今日重点”和“数据概览”工作台区块，数据仍来自 FastAPI 仪表盘接口。
  - 侧栏增加可滚动主导航区域、折叠态图标居中和中宽响应式网格，降低 1280x720 与 1059x736 下的溢出风险。

## 下一步建议

- 后端保持真实 API，继续用现有 `seed`、`doctor`、API smoke 和 pytest 验证。
- 后续如果继续参考 CordysCRM，应优先打磨“销售线索 -> 商机 -> 订单”的页面联动体验，而不是继续新增后台模块。
